  ---
  1. Full Exam Submission Persistence

  Every time a candidate clicks "Finish Test", a complete submission snapshot is
   saved to exam_sessions.submitted_snapshot (JSONB). This includes:
  - All MCQ answers (selected option, flagged, time per question)
  - All essay answers (full text, word count, time spent)
  - Complete question data (text, options, correct answers)
  - Exam config (title, type, duration, MCQ/essay counts)
  - Candidate info (name, userId, station number, center ID)
  - Timing (started at, submitted at, time remaining)

  This is a frozen record. Even if the question bank changes or auto-save gets
  corrupted, you can re-grade from this snapshot.

  Additionally, full_questions JSONB stores the complete question data at
  session creation time ‚ÄĒ so questions are always available for re-evaluation.

  2. Session Recovery (Continue After Crash)

  How it works:
  1. Candidate is on Station 5, taking the exam. Auto-save runs every 60
  seconds, saving answers + timer + question index
  2. System crashes / power goes out / browser closes
  3. Candidate (or proctor) opens any station with the same URL:
  costudy.in/exam?center=SESSION_ID&station=5 (or even a different station like
  station=12)
  4. The system checks: "Does this user have an IN_PROGRESS exam in this test
  center session?"
  5. If yes ‚Üí restores everything: all answers, current question position, exact
   time remaining
  6. Candidate continues exactly where they left off

  Works across stations ‚ÄĒ if Station 5 dies, just log in on Station 12 and the
  exam picks up seamlessly.
